# UI 테스트 체크리스트

## 목적

수동 QA, 릴리즈 직전 smoke 확인, Playwright 신규 시나리오 후보 선정을 같은 기준으로 수행하기 위한 최소 체크리스트입니다.

## 공통 준비

- 앱 실행: `npm run dev`
- 기본 라우트: `/#/`, `/#/wizard`, `/#/cart`, `/#/download`, `/#/history`, `/#/settings`
- 권장 브라우저 크기: 데스크톱 1440px 이상, 보조 확인용 모바일 폭 390px
- 기본 확인 항목: 한글 문구 노출, 버튼 비활성/활성 전환, 저장 후 재진입 복원, 오류 메시지 가시성

## 핵심 체크리스트

| 체크리스트 ID | 대응 상세 케이스 | 시나리오 | 우선순위 | 기대 결과 |
| --- | --- | --- | --- | --- |
| UI-CHK-01 | `UI-WIZ-001` | 홈에서 패키지 타입을 선택해 위저드로 진입 | P0 | 선택한 타입 기준으로 위저드가 열리고 이전 타입 상태가 섞이지 않는다. |
| UI-CHK-02 | `UI-WIZ-002` | `pip` 또는 `npm` 패키지 검색 후 장바구니 추가 | P0 | 검색 결과, 버전 선택, 장바구니 반영이 정상 동작한다. |

| UI-CHK-07 | `UI-HIS-001` | 이메일 전달 이력에서 재다운로드 | P0 | 히스토리의 수신자는 복원되지만 전역 설정 수신자는 덮어쓰지 않는다. |

| UI-CHK-09 | `UI-DL-003` | 빈 장바구니 상태에서 다운로드 페이지 진입 | P1 | 빈 상태 안내와 복귀 액션이 보인다. |
| UI-CHK-10 | `UI-DL-004` | 다운로드 취소 또는 실패 후 재시도 | P1 | 상태가 일관되게 갱신되고 재시도는 실패 항목만 대상으로 동작한다. |

## 권장 실행 순서

1. `UI-CHK-05`, `UI-CHK-06`으로 설정 화면과 SMTP 경로를 먼저 고정합니다.
2. `UI-CHK-01`, `UI-CHK-02`, `UI-CHK-03`, `UI-CHK-04`로 일반 패키지 다운로드 플로우를 확인합니다.
3. `UI-CHK-07`, `UI-CHK-08`로 회귀 가능성이 높은 분기 플로우를 확인합니다.
4. 릴리즈 직전에는 `UI-CHK-09`부터 `UI-CHK-12`까지 예외 흐름을 추가합니다.

## 자동화 연계

- 현재 Playwright 회귀 세트와 직접 매핑되는 항목: `UI-CHK-03`, `UI-CHK-05`, `UI-CHK-06(성공 경로)`, `UI-CHK-07`, `UI-CHK-08`, `UI-CHK-10`, `UI-CHK-11`
- 자동화 후보 우선순위가 높은 항목: `UI-CHK-01`, `UI-CHK-02`, `UI-CHK-04`
- 수동 확인 비중이 더 큰 항목: `UI-CHK-11`, `UI-CHK-12`

세부 절차와 실패 기준은 [UI 상세 테스트 케이스](./ui-testing-test-cases.md), Playwright 전환 설계는 [UI Playwright 전환 시나리오](./ui-testing-playwright-conversion.md)를 참고합니다.
